Portfolio – Reflectie WPL1

Situering binnen Profiel DVO

Waar sta ik nu?

Ik sta aan het begin van mijn ontwikkeling als programmeur/designer. Ik begrijp de basis van HTML, CSS en JavaScript en kan eenvoudige websites structureren en opbouwen.

Wat kan ik al?

Ik kan een website opbouwen met een duidelijke structuur, gebruik maken van semantische HTML en eenvoudige interactiviteit toevoegen met JavaScript. Ook kan ik een aanzet maken naar een mooi basisdesign voor een website.

Wat wil ik nog verbeteren?

Ik wil vooral mijn skills in talen zoals JavaScript en C# verder ontwikkelen. Daarnaast wil ik ook mijn grafische skills verbeteren om meer professionele ontwerpen te creëren.

Eindreflectie WPL1

Wat ging goed?

Het zelfstandig werken aan mijn portfolio ging goed. Ik heb geleerd om mijn code beter te organiseren en stap voor stap te werken.

Wat was moeilijk?

Het structureren van grotere projecten en het correct gebruiken van GitHub was in het begin moeilijk. Soms liep ik vast door kleine fouten in mijn code. Ook het werken in groep was voor mij niet eenvoudig, omdat ik het liefst alleen werk en de samenwerking niet altijd even vlot verliep.

Wat heb ik geleerd?

Ik heb geleerd dat fouten maken een belangrijk deel is van het leerproces. Door te blijven testen en op te zoeken begrijp ik mijn werk beter. Ook het opvolgen van deadlines en het plannen van mijn werk zijn vaardigheden die ik heb verbeterd.

Rollen binnen Profiel DVO

Rol 1: Designer

Als designer sta ik nog aan het begin van mijn ontwikkeling, maar ik beheers al de basisprincipes van visuele vormgeving. Ik kan een eenvoudige maar duidelijke layout ontwerpen en rekening houden met hiërarchie, spacing en leesbaarheid. In projecten zoals Huis Van Tichelen heb ik gewerkt met Adobe Illustrator, Photoshop en InDesign, waar ik leerde hoe een visuele stijl consequent toegepast wordt over verschillende dragers.

Ik wil mij verder verbeteren in het maken van meer professionele en verfijnde ontwerpen, en beter leren werken met kleurgebruik, typografie en compositie.

Rol 2: Developer / Programmeur

Als programmeur voel ik mij momenteel het minst sterk. Ik kan websites structureren met semantische HTML, styling toepassen met CSS en basisinteractiviteit bouwen met JavaScript. In projecten zoals mijn TakeAway-website heb ik zelf de volledige structuur, styling en JavaScript-logica uitgewerkt, met hulp van het internet en AI.

Daarnaast werk ik aan een persoonlijk project (Shadow of Nytherion), waarin ik met C#, Unity en Blender mijn technische vaardigheden verder ontwikkel. Ik wil mij verder verdiepen in JavaScript en C# en leren hoe ik grotere projecten beter kan structureren, eventueel via zelfstudie naast de opleiding.

Rol 3: Communicator

Communicatie is voor mij een belangrijk leerpunt. Ik kan mijn projecten schriftelijk duidelijk toelichten en uitleggen welke keuzes ik gemaakt heb. Tijdens WPL1 heb ik geleerd hoe belangrijk het is om mijn werk goed te documenteren, bijvoorbeeld via GitHub en mijn portfolio.

Mondeling communiceren en samenwerken in groep blijft voor mij uitdagend, maar ik probeer hier stap voor stap in te groeien door feedback te verwerken en open te blijven communiceren.

Rol 4: Professional

Als professional heb ik geleerd om zelfstandiger te werken en verantwoordelijkheid te nemen voor mijn eigen leerproces. Ik heb geleerd om met deadlines om te gaan, mijn werk beter te plannen en door te zetten wanneer iets niet meteen lukt.

Ik merk dat ik het best functioneer wanneer ik structuur heb en zelfstandig kan werken. In de toekomst wil ik mijn planning verder verbeteren en mijn werk consistenter opvolgen.

AIM-reflectie

Aandacht

Tijdens WPL1 heb ik geleerd om mijn aandacht beter te verdelen over grotere opdrachten. In het begin verloor ik soms het overzicht, maar door stap voor stap te werken lukte het steeds beter om mijn focus te bewaren.

Inzet

Ik heb mij ingezet om mijn portfolio en opdrachten zelfstandig af te werken. Wanneer ik vastliep, probeerde ik problemen zelf op te lossen door te testen, op te zoeken of hulp te vragen wanneer nodig.

Mindset

Mijn mindset is geëvolueerd van “alles moet meteen werken” naar “fouten maken hoort bij het leerproces”. Ik begrijp nu beter dat fouten een noodzakelijk onderdeel zijn van leren programmeren en ontwerpen.

Wat neem ik mee naar WPL2?

  • Een betere basis in HTML, CSS en JavaScript
  • Het belang van structuur in code en projecten
  • Het inzicht dat planning en documentatie essentieel zijn
  • Meer zelfvertrouwen in mijn technische vaardigheden

In WPL2 wil ik meer inzetten op samenwerking, betere communicatie en het verder professionaliseren van mijn werk.

Reflectie over X-factor

Mijn X-factor ligt in mijn doorzettingsvermogen en technische nieuwsgierigheid. Ik werk vaak aan eigen projecten naast school, zoals game development, om mezelf uit te dagen en bij te leren. Wanneer iets mij interesseert, kan ik er diep op ingaan en blijf ik zoeken tot ik het begrijp.

Deze motivatie en leergierigheid wil ik verder benutten in mijn opleiding en toekomstige werkplek.
